# API Reference

SALLY provides a powerful REST API for route planning, HOS compliance, and fleet management.

## Base URL

```
https://sally-api.apps.appshore.in/api/v1
```

## Quick Start

1. [Get your API key](/getting-started/api-keys)
2. [Authenticate your requests](/api-reference/authentication)
3. [Explore endpoints](/api-playground) in the interactive playground

## Core Endpoints

### Routes
Plan optimized, HOS-compliant routes with automatic rest and fuel stop insertion.

**Key endpoints:**
- `POST /api/v1/routes/plan` - Plan a new route
- `GET /api/v1/routes/{id}` - Get route details
- `POST /api/v1/routes/{id}/update` - Update route with triggers

[Explore in playground →](/api-playground#tag/Routes)

### Alerts
Manage dispatcher alerts and notifications for proactive fleet management.

**Key endpoints:**
- `GET /api/v1/alerts` - List active alerts
- `POST /api/v1/alerts/{id}/acknowledge` - Acknowledge alert
- `POST /api/v1/alerts/{id}/resolve` - Resolve alert

[Explore in playground →](/api-playground#tag/Alerts)

### HOS Validation
Validate Hours of Service compliance and ensure routes meet FMCSA regulations.

**Key endpoints:**
- `POST /api/v1/hos/validate` - Validate HOS compliance
- `GET /api/v1/hos/limits` - Get current HOS limits

[Explore in playground →](/api-playground#tag/HOS)

## Response Format

All API responses follow this structure:

### Success (200 OK)
```json
{
  "data": {
    // Response data here
  }
}
```

### Error (4xx, 5xx)
```json
{
  "statusCode": 400,
  "message": "Validation failed",
  "error": "Bad Request"
}
```

## Rate Limits

- **Staging keys**: 1,000 requests/hour
- **Production keys**: 10,000 requests/hour

Rate limit headers:
- `X-RateLimit-Limit` - Total requests allowed
- `X-RateLimit-Remaining` - Requests remaining
- `X-RateLimit-Reset` - Reset timestamp

## Status Codes

| Code | Description |
|------|-------------|
| 200 | Success |
| 201 | Created |
| 400 | Bad Request |
| 401 | Unauthorized |
| 404 | Not Found |
| 429 | Rate Limit Exceeded |
| 500 | Server Error |
