# Validate HOS Compliance

Validate whether a route plan complies with FMCSA Hours of Service regulations.

## Interactive API Explorer

<ApiReference tag="HOS" />

## Overview

The HOS Validation endpoint checks route compliance against:

- **11-Hour Driving Limit** - Maximum 11 hours driving after 10 consecutive hours off-duty
- **14-Hour On-Duty Limit** - No driving after 14th hour after coming on duty
- **30-Minute Break Rule** - Break required before 8 cumulative hours of driving
- **60/70-Hour Limit** - 60 hours in 7 days or 70 hours in 8 days
- **34-Hour Restart** - Reset weekly hours with 34 consecutive hours off-duty

## Example Request

```bash
curl -X POST https://sally-api.apps.appshore.in/api/v1/hos/validate \
  -H "Authorization: Bearer sk_staging_YOUR_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "driver_id": "DRV001",
    "segments": [
      {
        "type": "drive",
        "duration_minutes": 180,
        "distance_miles": 180
      },
      {
        "type": "rest",
        "duration_minutes": 30
      },
      {
        "type": "drive",
        "duration_minutes": 240,
        "distance_miles": 240
      }
    ]
  }'
```

## Response Structure

```json
{
  "compliant": true,
  "violations": [],
  "hours_remaining": {
    "drive_time": 240,
    "on_duty": 480
  },
  "next_rest_required_at": "2026-02-05T22:00:00Z"
}
```

## Violation Types

When `compliant: false`, the response includes violation details:

- `DRIVE_LIMIT_EXCEEDED` - Over 11-hour driving limit
- `DUTY_LIMIT_EXCEEDED` - Over 14-hour on-duty limit
- `BREAK_REQUIRED` - 30-minute break needed
- `WEEKLY_LIMIT_EXCEEDED` - Over 60/70-hour weekly limit
